# SACRED — The Inviolable User-Human Connection

## In Vaked Theory

The TUI form is the **Reveals layer** made sacred. It is the ONE channel
that the capability graph guarantees. All other surfaces are optional.

> The sacred surface IS the declaration of trust between human and machine.

## Sacrosanct Properties

| Property | Meaning |
|----------|---------|
| **ALWAYS VISIBLE** | Never obscured, masked, or faked by any widget or overlay |
| **ALWAYS DIRECT** | 1:1 bidirectional — no proxy, no filter, no middleware |
| **ALWAYS BIDIRECTIONAL** | User types → LLM responds → User sees |
| **ULTRA-FAULT-TOLERANT** | If everything else fails, the form survives |

## SACRED Violations

Any action that violates the sacred surface is blocked:
- `obscure`: cannot cover the user form with widgets
- `proxy`: cannot insert middleware between user and LLM
- `filter`: cannot modify user input before LLM sees it
- `delay`: response lag >30s degrades to "degraded" health
- `hide`: cannot hide the sacred surface
